# Legal and commercial readiness checklist

> **Commercial redistribution notice:** no repository-level software licence or
> ownership grant is currently present. Commercial redistribution, source
> transfer, sublicensing, or customer delivery is not authorized by this
> repository until the owner and qualified legal counsel approve and record the
> applicable rights.

This checklist records decisions that source code cannot make. It is not legal
advice and does not select a licence.

- [ ] Repository owner approves a software licence and delivery model.
- [ ] Code ownership and employee/contractor contributor assignments are
  documented.
- [ ] Third-party Python, npm, container-image, and build-tool licences are
  inventoried and reviewed from the release SBOM.
- [ ] FK SOLUTIONS name, monogram, login image, icons, and other visual assets
  have documented ownership or licensed use.
- [ ] Font use is approved; the current UI distributes no font files and uses a
  system stack.
- [ ] Demo/test datasets and synthetic model inputs have documented provenance.
- [ ] Customer-supplied datasets, prompts, documents, predictions, model
  artefacts, and telemetry have defined ownership and permitted-use terms.
- [ ] Privacy roles, lawful basis, retention, deletion, export, incident
  notification, subprocessors, and cross-border transfer responsibilities are
  assigned.
- [ ] Backup, log, trace, model, citation, and audit retention obligations are
  reconciled with customer data terms.
- [ ] Open-source notices and required attribution are included in delivery
  materials.
- [ ] Export-control, sanctions, encryption, and model-use restrictions are
  classified where applicable to the customer and delivery jurisdictions.
- [ ] Security claims, availability claims, support terms, warranties,
  indemnities, and limitation-of-liability language are reviewed.
- [ ] The controlled-pilot scope and explicit exclusions are incorporated into
  the customer acceptance document.
